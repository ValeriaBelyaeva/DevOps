# Отчёт по лабораторной: Bad vs Good CI/CD

## 1. Цель работы
Сравнить "плохой" и "хороший" CI/CD workflow: выявить минимум 3 bad practices, исправить их и показать эффект исправлений.


## 2. Реализация
### 2.1 Плохой workflow
Файл: `.github/workflows/bad-ci.yml`

Ключевые проблемы:
- floating refs у actions
- избыточные permissions
- агрессивный debug (риск утечек)
- тесты не блокируют пайплайн
- деплой запускается на PR/любых событиях
- отсутствие кэширования

### 2.2 Хороший workflow
Файл: `.github/workflows/good-ci.yml`

Исправления:
- стабильные версии actions
- least privilege permissions
- concurrency + timeout
- встроенное кеширование pip
- деплой только из main и только после успешных тестов
- environment для возможности protection rules

## 6. Вывод
"Хороший" workflow стал:
- безопаснее (least privilege, меньше утечек в логах, деплой ограничен),
- стабильнее (фиксированные версии и явная версия Python),
- быстрее (кэширование pip),
- качественнее (тесты реально блокируют деплой).
